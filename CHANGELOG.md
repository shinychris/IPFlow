## [2.0.0] - 2026-02-28

### 🎉 重大更新

IPFlow v2.0 正式发布！这是一个里程碑版本，系统从单机应用升级为支持多租户的 SaaS 平台，并新增完整的专利和商标申请支持。

### ✨ 新增功能

#### 多租户与组织
- 完整的组织管理系统
- 多成员协作支持
- 基于角色的权限控制（RBAC）
  - 超级管理员
  - 组织管理员
  - 项目经理
  - 普通成员
  - 只读用户
- 组织邀请机制（邀请链接）
- 组织级别资源隔离
- 租户上下文中间件

#### 订阅与计费
- 分级订阅计划
  - Free: 3项目/100MB/1成员
  - Starter: 10项目/1GB/3成员
  - Professional: 50项目/10GB/10成员
  - Enterprise: 无限/100GB/无限成员
  - Agency: 500项目/1TB/20成员
- 资源配额管理（项目数/存储空间/成员数）
- 限额控制服务（QuotaService）
- 使用统计与监控
- 发票管理系统
- 支付提供商集成框架（预留 Webhook）

#### 管理后台
- 超级管理员仪表盘（统计概览）
- 用户管理（列表、详情、更新、删除、搜索）
- 组织管理（审核、限额调整）
- 计划管理（动态配置）
- 审计日志查询

#### 审计与监控
- 完整的操作审计日志（AuditLog）
- 请求追踪（X-Request-ID）
- 健康检查端点（/health、/health/simple）
- 系统指标收集
- 资源使用统计

### 🔬 专利申请支持（新增）

支持三种专利类型：
- 发明专利
- 实用新型
- 外观设计

功能包括：
- 专利信息管理（技术领域、背景技术、发明内容等）
- 权利要求书结构化编辑
- 说明书分段编辑（技术领域、背景技术、技术问题、技术方案、有益效果、附图说明）
- 专利附图上传与管理
- 导出功能（生成申请文件包）

### 🏷️ 商标注册支持（新增）

支持六种商标类型：
- 文字商标
- 图形商标
- 组合商标
- 三维商标
- 声音商标
- 颜色商标

功能包括：
- 商标信息管理
- 尼斯分类完整支持（45个类别）
- 商标图样上传
- 证明材料管理
- 导出功能

### 📝 软件著作权优化

- 代码处理引擎增强
  - 支持 40+ 编程语言
  - 自动识别代码文件
  - 智能过滤（node_modules、.git 等）
- 前30页+后30页分页生成
- 行号自动添加（5位对齐）
- 说明书模板支持（5种模板）
- 合规检查自动校验
- 导出功能（ZIP 材料包）

### 🔧 技术改进

#### 后端
- 技术栈重构：Node.js + Express → Python + FastAPI
- ORM 升级：Drizzle → SQLModel
- 认证升级：Session → JWT + Redis
- 异步架构优化
- 多租户上下文管理
- 限流与安全防护
- 结构化日志（JSON 格式）
- 全局异常处理

#### 前端
- 状态管理升级：Redux → Zustand
- 新增 TanStack Query 服务端状态管理
- TypeScript 类型系统完善
- 组件库升级至 shadcn/ui
- 响应式设计优化
- 新增代码预览组件（语法高亮、行号、分页）
- 新增富文本编辑器
- 新增文件上传组件（拖拽、进度）

#### 数据库
- 新增订阅相关表（plan、subscription、invoice）
- 新增审计日志表（audit_log）
- 新增专利相关表（patent_data、patent_claim、patent_drawing）
- 新增商标相关表（trademark_data、nice_classification、trademark_nice_class）
- 组织表结构优化
- 索引优化

## 版本说明

### 版本号格式

`MAJOR.MINOR.PATCH`

- **MAJOR**: 不兼容的 API 修改
- **MINOR**: 向下兼容的功能新增
- **PATCH**: 向下兼容的问题修复

### 版本生命周期

| 版本 | 状态 | 支持时间 |
|------|------|----------|
| 2.0.x | 活跃开发 | 当前 |
| 1.5.x | 维护模式 | 2024-06-30 前 |
| 1.0.x | 已停止 | 不再支持 |

### 升级指南

#### 从 1.x 升级到 2.0

1. 备份现有数据
2. 更新代码到 v2.0 分支
3. 执行数据库迁移：`alembic upgrade head`
4. 更新环境变量配置
5. 重新部署应用

> ⚠️ 注意：v2.0 有重大架构变更，建议在测试环境验证后再升级生产环境。
