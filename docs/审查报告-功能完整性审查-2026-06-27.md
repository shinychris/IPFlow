## 二、总体结论

**IPFlow 的核心定位——「软著/专利/商标申报材料辅助生成、整理、合规检查与导出」——的三条主线业务闭环已完整实现并可用。** 三种业务类型的「创建项目 → 录入信息 → 上传/拉取材料 → AI 生成草稿 → 预览 → 合规检查 → 导出」全流程在前端均有页面、在后端均有 API 与服务实现。

但审查发现以下**必须整改**的问题（按优先级）：

- 🔴 **P0 测试无法收集/运行**：3 个测试文件因导入缺失符号/过时 API 而报错，导致测试套件无法整体运行。
- 🟠 **P1 设置页保存功能未接线**：「保存更改」「保存 AI 配置」按钮无 onClick，用户无法持久化设置。
- 🟠 **P1 合规检查仅软著有真实实现**：专利、商标合规为前端占位空实现，后端无对应路由。
- 🟡 **P2 配套页面缺失**：订阅管理、组织管理、管理后台三个模块后端 API + 前端 store 齐备，但前端无页面（store 为死代码）。
- 🟡 **P2 字段命名不一致**：后端返回 snake_case，shared 类型部分用 camelCase，前端靠大量 `as any` 绕过类型。

以下为按优先级可暂不整改的「已知限制」（标注为 ℹ️，非功能缺失）：

- ℹ️ 支付为沙箱 Mock（无真实微信/支付宝/Stripe Provider 集成）—— AGENTS.md 已声明订阅/支付为「配套能力」。
- ℹ️ 导出为 TXT+ZIP（无 DOCX/PDF）—— 可后续增强。
- ℹ️ AI 草稿默认走 template，claude_code 后端需专用部署机配置—— 设计如此。
- ℹ️ JWT 登出未强制 Redis 黑名单—— 安全增强项。
