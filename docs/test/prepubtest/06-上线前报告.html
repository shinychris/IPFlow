发现的问题

#3 【严重】导出源代码换行符被转义为字面\n,代码无法按页50行展示 严重 数据

预期: 02_源代码_鉴别材料.txt 每行应为真实换行(0x0a),每页50行带行号,符合软著打印规范

实际: 导出文件中行间分隔符为字面'\n'(0x5c 0x6e)而非真实换行; 整个源代码实际为单行含字面\n文本,破坏'每页50行带行号'格式; 根因 code_processor.py:402 separator=f'\n//...', :443/:457/:472 page_content='\n'.join(...) 全部误用双反斜杠; pages_data入库即含字面\n,导出仅原样写出

证据: xxd artifacts/copyright_unzipped/02_源代码_鉴别材料.txt 偏移0xc0见5c6e; code_processor.py:402,443,457,472; 这是软著申请核心交付物,直接影响材料可用性

#5 【重要】尼斯分类全局查询路由被错误挂到projects/{project_id}前缀下 严重 功能

预期: GET /api/v1/trademarks/nice-classes 为全局端点(用户创建商标前浏览分类),应可直接访问

实际: GET /trademarks/nice-classes 返回404; 因 trademarks/__init__.py:16 把 nice_classification_router 整个挂到 prefix=/projects/{project_id} 下, 导致全局 /nice-classes 与 /nice-classes/by-id/{id} 都需 project_id; 用户在创建商标项目前无法浏览分类,严重影响易用性

证据: GET /api/v1/trademarks/nice-classes -> 404; GET /api/v1/trademarks/projects/{id}/nice-classes -> 200 total=45; trademarks/__init__.py:16

#6 【重要】GET /health 完整健康检查500(import async_session_maker失败) 严重 功能

预期: /health 应返回各依赖(db/redis/storage)探测状态

实际: /health 返回 INTERNAL_ERROR 500; 根因 main.py:128 'from ipflow.db import async_session_maker' 但 db模块仅暴露 AsyncSessionLocal(非async_session_maker), ImportError; 健康检查不可用,影响运维监控

证据: backend log traceback: ImportError: cannot import name 'async_session_maker'; main.py:128; db/session.py 只定义 AsyncSessionLocal

#1 重复注册同邮箱返回422而非409 轻微 异常

预期: 409 Conflict 或 400

实际: 422 VALIDATION_ERROR msg=该邮箱已被注册

证据: results/07-register-dupe.json；语义正确(已拦截),仅状态码非RESTful惯例,轻微

#2 说明书page_count/word_count由服务端重算(非客户端值) 轻微 数据

预期: PUT 提交 page_count=18 时可能被采用

实际: 服务端忽略客户端page_count,按 content_html 纯文本字数重算: word_count=len(text), page_count=max(1,word_count//800) (manuals.py:324-325); 属设计决策(防伪造),非bug,但需文档说明

证据: results/15-compliance.txt; manuals.py:324

#4 专利导出同时生成DOCX版本(超出预期,加分项) 建议 功能

预期: PRD只要求ZIP+TXT

实际: 专利ZIP含 说明书.docx + 权利要求书.docx (python-docx可用时), 内容与TXT一致; 文档化处理完善

证据: artifacts/patent_unzipped/ 含4文件
