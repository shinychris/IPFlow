# IPFlow 上线前测试 — 功能清单（Feature Inventory）

> 依据后端 `/openapi.json`（100 个路由）与源码梳理。本清单是测试覆盖的「索引」，每项对应用户故事与用例。
> 标注：🔴=P0 核心必过  🟠=P1 重要  🟡=P2 一般  ⚪=配套不深入

---

## 一、认证模块（Auth） `/api/v1/auth/*`

| ID | 功能 | 方法 路径 | 优先级 |
|----|------|----------|-------|
| F-AUTH-1 | 用户注册（自动登录，返回 token） | POST `/auth/register` | 🔴 |
| F-AUTH-2 | 登录（用户名或邮箱） | POST `/auth/login` | 🔴 |
| F-AUTH-3 | 当前用户信息 | GET `/auth/me` | 🟠 |
| F-AUTH-4 | 刷新 token | POST `/auth/refresh` | 🟠 |
| F-AUTH-5 | 登出（token 加黑名单） | POST `/auth/logout` | 🟠 |
| F-AUTH-6 | 邮箱验证（非阻断） | POST `/auth/verify-email` | 🟡 |
| F-AUTH-7 | 忘记密码/重置密码 | POST `/auth/forgot-password` `/reset-password` | 🟡 |
| F-AUTH-8 | 受保护端点鉴权（401 校验） | （各端点） | 🔴 |

## 二、项目管理（Projects） `/api/v1/projects`

| ID | 功能 | 方法 路径 | 优先级 |
|----|------|----------|-------|
| F-PROJ-1 | 创建项目（指定类型） | POST `/projects` | 🔴 |
| F-PROJ-2 | 项目列表（按 owner 隔离） | GET `/projects` | 🔴 |
| F-PROJ-3 | 项目详情 | GET `/projects/{id}` | 🟠 |
| F-PROJ-4 | 更新项目（含完成日期等） | PATCH `/projects/{id}` | 🟠 |
| F-PROJ-5 | 软删除项目 | DELETE `/projects/{id}` | 🟡 |
| F-PROJ-6 | 复制项目 | POST `/projects/{id}/duplicate` | 🟡 |

## 三、软著（Copyright） `/api/v1/copyright/*`

| ID | 功能 | 方法 路径 | 优先级 |
|----|------|----------|-------|
| F-CR-1 | 软件信息 读/写 | GET/PUT `/copyright/projects/{id}/software-info` | 🔴 |
| F-CR-2 | 上传代码 ZIP 并处理（前30+后30页） | POST `/copyright/projects/{id}/code-bundles` | 🔴 |
| F-CR-3 | 代码包列表/详情/预览/删除 | GET `/copyright/projects/{id}/code-bundles[/{bid}[/preview]]` | 🟠 |
| F-CR-4 | 操作说明书 增改查删 | POST/PUT/GET/DELETE `/copyright/projects/{id}/manuals` | 🔴 |
| F-CR-5 | AI 草稿生成任务（template 后端） | POST `/copyright/generation-jobs` | 🟡 |
| F-CR-6 | 导出任务创建 | POST `/copyright/projects/{id}/export-jobs` | 🔴 |
| F-CR-7 | 导出任务查询 | GET `/copyright/export-jobs/{id}` | 🔴 |
| F-CR-8 | **导出制品下载（ZIP）** | GET `/copyright/export-jobs/{id}/download` | 🔴 |
