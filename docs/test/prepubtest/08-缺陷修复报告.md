## 二、严重缺陷修复详情

### ✅ 缺陷 #3：软著导出源代码换行被转义为字面 `\n`

**根因**：`backend/src/ipflow/services/copyright/code_processor.py` 中 4 处误用双反斜杠转义：
- `merge_code_files` 的文件分隔符（原 L402）：`f"\\n// ====...\\n"` → `f"\n// ====...\n"`
- `paginate_code_front_back_30` 的页面内容连接（原 L443/457/472）：`'\\n'.join(...)` → `'\n'.join(...)`

**改动文件**：`backend/src/ipflow/services/copyright/code_processor.py`

**验证证据**（真实导出 ZIP 二进制校验，`artifacts/copyright_export_FIXED.zip`）：
```
02_源代码_鉴别材料.txt:
  字面 \n (5c6e) 次数: 0        ← 修复前为大量字面\n
  真实换行(0x0a) 次数: 3183     ← 修复后正确
  行号00001存在: True | 行号00050存在: True
  行号00050后应换到00051(每页50行): True
```
源代码现按「行号 + 真实换行」正确分页，符合软著「每页50行带行号、连续打印」规范。

**回归测试**：新增 2 个单测防止回归（见第四节）。

---

### ✅ 缺陷 #5：尼斯分类全局查询路由被错误挂到 `projects/{project_id}` 前缀

**根因**：`backend/src/ipflow/api/v1/trademarks/__init__.py:16` 把整个 `nice_classification_router`（含全局 `/nice-classes`、`/nice-classes/by-id/{id}` 与项目级 `/nice-classes/selected`）统一挂到 `prefix="/projects/{project_id}"` 下，导致全局端点需带 `project_id`，用户创建商标前无法浏览分类。

**改动文件**：
- `backend/src/ipflow/api/v1/trademarks/nice_classification.py`：拆出 `global_router`（承载 `/nice-classes`、`/nice-classes/by-id/{id}`），保留 `router`（承载项目级 `/nice-classes/selected*`）。
- `backend/src/ipflow/api/v1/trademarks/__init__.py`：`global_router` 以无前缀挂载，`router` 仍挂 `/projects/{project_id}`。

**验证证据**：
```
GET /api/v1/trademarks/nice-classes?page_size=100  → HTTP 200, total=45（无需 project_id）
项目级端点不受影响：
  GET /api/v1/trademarks/projects/{id}/nice-classes/selected → 仍正常
```

## 三、轻微缺陷修复详情

### ✅ 缺陷 #1：重复注册同邮箱返回 422 → 改为 409

**根因**：`auth.py` 重复邮箱/用户名检查用 `ValidationException`（固定 422），语义不符（数据合法但与现有资源冲突，应为 409 Conflict）。

**改动文件**：
- `backend/src/ipflow/core/exceptions.py`：新增 `ConflictException`（code=`CONFLICT`，HTTP 409），沿用其他异常的模式。
- `backend/src/ipflow/api/v1/auth.py`：注册（邮箱/用户名重复）与 profile 更新（邮箱冲突）3 处改用 `ConflictException`。

**验证证据**：重复邮箱注册 → `HTTP 409`，`{"code":"CONFLICT","message":"该邮箱已被注册"}`。

---

### ✅ 缺陷 #2：说明书 page_count/word_count 服务端重算（文档化）

**说明**：`ManualUpdate` 接受客户端 `word_count`/`page_count`，但处理器始终按 `content_html` 纯文本**重算**（`manuals.py`：`word_count=len(去标签文本)`，`page_count=max(1,word_count//800)`）。这是**防伪造、保证合规一致**的设计决策，非 bug。

**改动文件**：`backend/src/ipflow/api/v1/copyright/manuals.py`：在 `ManualUpdate` docstring 与字段注释中明确说明「客户端值将被忽略，服务端按 content_html 重算」，消除前端困惑。
