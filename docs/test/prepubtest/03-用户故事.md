# IPFlow 上线前测试 — 用户故事（User Stories）

> 每个故事的验收标准（Then）均包含**数据预期**，作为后续数据校验的依据。
> 优先级：P0 核心·必过 / P1 重要 / P2 一般

---

## US-1 访客注册并登录系统
**作为** 一个未注册的访客
**我想要** 注册账号并登录系统
**以便于** 开始准备我的知识产权申报材料

**验收标准**
- AC1（正常路径）
  - Given 一个未注册的访客
  - When 提交合法的邮箱/用户名/密码注册
  - Then 注册成功（201），返回 access_token；无需邮件验证即可用该 token 访问 `/auth/me`
- AC2（数据）
  - Given 已注册用户
  - When 用用户名或邮箱+密码登录
  - Then 登录成功，返回的 user_id 与注册时 id 一致；token_type=bearer
- AC3（异常：鉴权）
  - Given 无 token 或伪造 token
  - When 访问受保护端点
  - Then 返回 401

## US-2 个人开发者完成一套软著材料并导出（核心 P0）
**作为** 一个独立开发者
**我想要** 为我的软件项目准备软著申报材料（软件信息+源代码+说明书）并导出整套材料包
**以便于** 向版权中心提交软件著作权登记

**验收标准**
- AC1（业务流）
  - Given 已登录用户
  - When 创建软著项目 → 填写软件信息 → 上传代码 ZIP → 创建说明书 → 运行合规 → 创建导出任务 → 下载
  - Then 全链路成功，**真实下载到一个 ZIP 文件**
- AC2（数据：代码处理统计）
  - Given 一个含多语言代码文件的 ZIP
  - When 上传处理
  - Then 返回 total_files/total_lines/language_stats 正确；页数应为「前30页+后30页」（代码充足时最多60页）；每页 50 行带行号
- AC3（数据：导出内容完整性）
  - Given 已完成导出
  - When 解压下载的 ZIP
  - Then 含 01_软件信息 / 02_源代码鉴别材料 / 03_操作说明书 / 04_材料清单 / 05_打印指南 / 06_网报对照表；其中软件全称/版本号与填写一致
- AC4（异常：未上传代码直接导出）
  - Given 只有软件信息、无代码包
  - When 创建导出任务
  - Then 任务失败，error 提示需先上传源代码（不应 500 崩溃）
- AC5（数据：隔离）
  - Given A 用户的项目
  - When B 用户访问 A 的项目资源
  - Then 返回 404（按 owner_id 隔离）

## US-3 发明人完成一套专利材料并导出（核心 P0）
**作为** 一个发明人
**我想要** 撰写专利的说明书与权利要求书并导出
**以便于** 向专利局提交专利申请

**验收标准**
- AC1（业务流）
  - Given 已登录用户
  - When 创建专利项目 → 填写专利信息 → 添加权利要求 → 创建导出任务 → 下载
  - Then 全链路成功，真实下载到 ZIP，含「说明书」+「权利要求书」
- AC2（数据：权利要求一致性）
  - Given 添加了 N 条权利要求
  - When 导出并解压
  - Then 权利要求书文件中条目数 = N，编号/内容与提交一致
- AC3（异常：从属权利要求编号校验）
  - Given 一个 dependent 权利要求引用的编号 >= 自身编号
  - When 提交
  - Then 返回 400，不创建
- AC4（异常：无权利要求导出）
  - Given 只有专利信息、无权利要求
  - When 创建导出任务
  - Then 任务失败提示需至少一项权利要求

## US-4 品牌方完成一套商标材料并导出（核心 P0）
**作为** 一个品牌负责人
**我想要** 选择商标类别与尼斯分类并导出商标注册申请书
**以便于** 向商标局提交商标注册

**验收标准**
- AC1（业务流）
  - Given 已登录用户
  - When 创建商标项目 → 填写商标信息 → 查询尼斯分类 → 选择分类 → 创建导出任务 → 下载
  - Then 全链路成功，真实下载到 ZIP，含「商标注册申请书」
- AC2（数据：尼斯分类查询）
  - Given 系统（开发环境自动 seed 45 类）
  - When 查询尼斯分类
  - Then 返回 45 个类别（class_number 1-45），含 class_name
- AC3（数据：分类一致性）
  - Given 为商标选择了 K 个分类
  - When 导出并解压
  - Then 申请书中分类清单 = K 项，类别号/名称/商品服务与选择一致
- AC4（异常：未选分类导出）
  - Given 只有商标信息、无分类
  - When 创建导出任务
  - Then 任务失败提示需至少选择一个尼斯分类

## US-5 系统在外部依赖缺失时优雅降级（健壮性 P0）
**作为** 运维/上线把关者
**我想要** 系统在 Redis/MinIO/SMTP 不可用时不崩主流程
**以便于** 稳定服务核心业务

**验收标准**
- AC1（数据）
  - Given sim 环境（无 Redis/MinIO/SMTP）
  - When 注册、上传、导出
  - Then 主流程全部正常；token 黑名单/限流降级到内存；邮件发送失败不阻断注册

## US-6 用户管理多个不同类型的并行项目（数据一致性 P1）
**作为** 一个同时处理多种知识产权的用户
**我想要** 在同一账号下创建软著/专利/商标多个项目且互不干扰
**以便于** 集中管理

**验收标准**
- AC1（数据）
  - Given 同一用户
  - When 创建 copyright + patent + trademark 三个项目
  - Then 列表返回 3 个项目，类型各一；各项目资源（软件信息/专利信息/商标信息）按 project_type 正确隔离，不串数据
