# 软著申请助手设计指南

## 设计方法论

**选择：Ant Design System**
- 专为中文企业应用优化，后台管理场景最佳实践
- 表单密集型工作流的标准范式
- 数据展示与文档管理组件完善

核心原则：清晰、高效、可信赖的专业工具体验

---

## 布局系统

**整体结构：侧边栏导航 + 主内容区**
```
├─ 左侧固定导航栏 (宽度: 240px)
│  ├─ Logo区域
│  ├─ 项目切换器
│  ├─ 主导航菜单（折叠式分组）
│  └─ 用户信息/设置
│
└─ 主内容区 (右侧占满)
   ├─ 页面标题栏 (面包屑 + 标题 + 操作按钮)
   ├─ 内容容器 (max-width: 1200px, 居中)
   └─ 底部操作栏 (固定在视口底部，仅在表单/编辑页)
```

**间距规范：使用 Tailwind 的 4、8、16、24 单位**
- 卡片间距: `gap-6` (24px)
- 卡片内边距: `p-6` (24px)
- 表单项间距: `space-y-4` (16px)
- 页面边距: `px-6 py-8`

## 中文排版系统

**字体栈**
```
font-sans: 'PingFang SC', 'Microsoft YaHei', 'Helvetica Neue', Arial, sans-serif
font-mono: 'JetBrains Mono', 'Consolas', monospace (代码预览)
```

**字号层级**
- 页面主标题: text-2xl (24px) font-semibold
- 卡片/区块标题: text-lg (18px) font-medium
- 正文/表单标签: text-base (16px)
- 辅助文字/提示: text-sm (14px)
- 代码/行号: text-xs (12px) font-mono

**行高**
- 标题: leading-tight (1.25)
- 正文: leading-relaxed (1.625)
- 代码区域: leading-6 (固定行高，符合60页规范)

## 核心组件库

### A. 导航与布局
- **侧边栏菜单**: 双层折叠结构，活跃项高亮，图标+文字
- **顶部操作栏**: 面包屑导航 + 页面标题 + 主要操作按钮（右对齐）
- **步骤条**: 横向步骤指示器（5步流程：创建→信息→代码→文档→导出）

### B. 表单与输入
- **表单布局**: 左对齐标签（宽度120px），必填项红色星号
- **输入框**: 统一高度 h-10，圆角 rounded-md，清晰的聚焦状态
- **下拉选择器**: 带搜索功能，支持分组（主体类型、开发方式等）
- **文件上传**: 拖拽区域 + 文件列表（显示进度、大小、删除操作）
- **富文本编辑器**: 工具栏固定顶部，预览/编辑模式切换

### C. 数据展示
- **项目卡片**: 网格布局 grid-cols-3，显示项目名、版本、状态、时间
- **合规清单**: 可展开的检查项列表，图标指示（✓通过 / ⚠待补正 / ✗未完成）
- **文档预览器**: 左侧缩略图导航 + 右侧PDF渲染（页码显示）
- **代码查看器**: 行号 + 语法高亮 + 页眉预览（软件全称+版本号）

### D. 反馈与引导
- **进度指示**: 环形进度图（显示完成百分比）+ 待办事项计数
- **提示卡片**: 分级提醒（信息/警告/错误），带图标和操作链接
- **空状态**: 插画 + 引导文案 + 快速操作按钮
- **加载状态**: 骨架屏（表单）+ 旋转加载器（操作反馈）

### E. 操作与交互
- **主按钮**: 实心背景，用于主要操作（提交、生成、导出）
- **次按钮**: 轮廓样式，用于取消、预览、返回
- **文字按钮**: 用于列表内操作（编辑、删除、查看）
- **浮动操作栏**: 固定底部，包含"保存草稿"、"上一步"、"下一步"

## 关键页面布局

### 1. 项目列表（首页）
```
顶部工具栏: [搜索框] [筛选器] [新建项目按钮]
项目卡片网格: 3列，每卡片显示
  - 项目名称（大字）
  - 版本号 + 状态标签
  - 完成进度条
  - 更新时间
  - 快速操作菜单（…）
空状态: 居中大图标 + "创建第一个软著项目" 按钮
```

### 2. 五步向导流程
**统一结构**:
```
顶部: 步骤条（高亮当前步骤）
主区域: 
  左栏 (70%): 表单/编辑器/预览
  右栏 (30%): 
    - 实时检查清单
    - 规则说明卡片
    - 快捷操作链接
底部: 固定操作栏 [保存草稿] [上一步] [下一步/完成]
```

### 3. 代码处理页
```
上方: 文件上传区（拖拽框 + ZIP/Git连接选项）
下方分栏:
  左侧 (50%): 目录树（可勾选包含文件）
  右侧 (50%): 代码预览窗口
    - 显示抽取规则（前30页+后30页）
    - 行数统计实时更新
    - 格式预览（行号+页眉示例）
底部: [重新上传] [生成PDF预览] [确认使用]
```

### 4. 文档编辑页
```
工具栏: 模板选择器 + 格式工具 + 保存状态
主编辑区（双栏）:
  左侧: 章节大纲树（可拖拽排序）
  右侧: 富文本编辑器
    - 插入截图按钮（触发上传）
    - 自动图注编号
    - 字数/页数统计（实时）
右侧栏: 
  - 预览缩略图
  - 合规提示（≥15页，30行/页）
```

### 5. 导出打包页
```
顶部: 最终检查清单（全部通过 ✓ 才可导出）
中间: 文件树预览（展示ZIP包结构）
  ├─ 01_申请表_占位.pdf
  ├─ 02_源代码_鉴别材料.pdf
  ├─ 03_操作说明书.pdf
  ├─ 04_证明材料/
  └─ ...
底部: [导出ZIP包] [生成打印指南] [查看网报对照表]
```

## 视觉增强

**图标系统**: Ant Design Icons（中文场景优化）
**插图**: 空状态使用简洁线条插画，传达专业感

## 响应式策略
- 桌面优先（主场景 1440px+）
- 平板降级：侧边栏折叠为抽屉
- 手机端：仅支持查看与简单审阅（不支持完整编辑流）

---

## 无障碍
- 表单标签与输入框明确关联
- 键盘导航支持（Tab顺序、快捷键提示）
- 错误提示与字段绑定（aria-describedby）
- 高对比度文字确保可读性
