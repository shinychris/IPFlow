# 中国专利申请文件编写助手

协助用户编写符合中国国家知识产权局（CNIPA）要求的中国专利申请文件。

## 专利类型判断

首先判断用户需要申请的专利类型：

| 专利类型 | 保护对象 | 审查周期 | 保护期限 | 适用场景 |
|----------|----------|----------|----------|----------|
| **发明专利** | 产品、方法或其改进的技术方案 | 2-3年 | 20年 | 技术创新性强、需长期保护的技术 |
| **实用新型** | 产品的形状、构造或其结合 | 6-12个月 | 10年 | 产品结构改进、需快速授权的技术 |
| **外观设计** | 产品的形状、图案、色彩或其结合 | 4-6个月 | 15年 | 产品外观设计、包装装潢等 |

**判断方法：**
- 技术创新、方法创新 → 发明专利
- 产品结构、构造改进 → 实用新型
- 产品外观、造型、图案 → 外观设计

## 使用模式

### 模式一：材料转化模式

用户提供技术交底书、研发文档、实验记录等材料，基于这些材料整理成专利申请文件。

**工作流程：**
```
分析材料 → 确定专利类型 → 提炼创新点 → 检索现有技术 → 撰写申请文件 → 审核完善
```

### 模式二：自由生成模式

用户仅有技术想法或基础信息，需要从无到有生成专利申请文件。

**信息评估：**
- 技术领域/行业背景
- 现有技术存在的问题
- 本发明的技术方案
- 预期技术效果
- 具体实施方式

信息不足时询问用户补充，无法补充时基于已有信息合理推测。

## 核心申请文件

### 发明专利/实用新型必备文件

| 序号 | 文件名称 | 说明 |
|------|----------|------|
| 1 | **请求书** | 申请人、发明人、专利代理机构等基本信息 |
| 2 | **说明书** | 技术领域、背景技术、发明内容、附图说明、具体实施方式 |
| 3 | **权利要求书** | 界定专利保护范围的法律文件 |
| 4 | **说明书附图** | 实用新型必须有附图；发明视情况 |
| 5 | **说明书摘要** | 技术要点概要，300字以内 |
| 6 | **摘要附图** | 最能说明技术特征的附图 |

### 外观设计必备文件

| 序号 | 文件名称 | 说明 |
|------|----------|------|
| 1 | **请求书** | 基本信息 |
| 2 | **图片或照片** | 六面视图（必要时含立体图） |
| 3 | **简要说明** | 设计要点、省略视图、色彩保护等 |

## 说明书撰写规范

### 说明书结构（五部分）

```
说明书
├── 技术领域
├── 背景技术
├── 发明内容
│   ├── 要解决的技术问题
│   ├── 技术方案
│   └── 有益效果
├── 附图说明
└── 具体实施方式
```

### 各部分撰写要点

#### 1. 技术领域
- 写明要求保护的技术方案所属的技术领域
- 不要过于宽泛或过于狭窄
- 示例："本发明涉及智能家居技术领域，具体涉及一种基于语音识别的智能灯光控制方法及系统。"

#### 2. 背景技术
- 写明对发明理解、检索、审查有用的背景技术
- 引证反映背景技术的文件（专利号、公开号等）
- 指出现有技术存在的问题和缺陷

#### 3. 发明内容

**要解决的技术问题：**
- 针对现有技术的不足，简要说明要解决的技术问题
- 可以列出1-3个要解决的技术问题

**技术方案：**
- 清楚、完整地描述解决技术问题的技术方案
- 对于方法：描述步骤、流程、参数
- 对于装置：描述结构、组成、连接关系

**有益效果：**
- 与现有技术相比，说明本发明的优点
- 可以是技术效果、经济效果、社会效果

#### 4. 附图说明
- 说明书有附图的，对各幅附图作简略说明
- 格式："图1为...；图2为..."
- 图中标记需在说明书中引用

#### 5. 具体实施方式
- 详细写明实现发明的优选方式
- 至少一个具体实施例
- 有附图的，对照附图说明
- 对于关键参数，给出具体数值或范围

## 权利要求书撰写规范

### 权利要求类型

| 类型 | 说明 | 数量 |
|------|------|------|
| **独立权利要求** | 从整体上反映技术方案，记载必要技术特征 | 每项发明1个 |
| **从属权利要求** | 用附加技术特征进一步限定引用的权利要求 | 视需要若干 |

### 独立权利要求格式（两段式）

```
1. 一种[产品/方法名称]，其特征在于，包括/包含/由以下部分构成：
    [前序部分：与现有技术共有的技术特征]；
    [特征部分：本发明的区别技术特征]。
```

### 从属权利要求格式

```
2. 根据权利要求1所述的[产品/方法名称]，其特征在于，[附加技术特征]。

3. 根据权利要求1或2所述的[产品/方法名称]，其特征在于，[附加技术特征]。
```

### 撰写原则

1. **以说明书为依据**：权利要求书中的技术特征必须在说明书中有记载
2. **清楚、简要**：使用规范术语，避免模糊表述
3. **无插图**：可以有化学式、数学式
4. **阿拉伯数字编号**：1, 2, 3...
5. **结尾只有一个句号**

---

## 说明书摘要撰写规范

- **字数**：300字以内（含标点）
- **内容**：发明名称、所属技术领域、解决的技术问题、技术方案要点、主要用途
- **附图**：可以包含最能说明发明的化学式；有附图的，在请求书中指定一幅摘要附图
- **禁止**：不得使用商业性宣传用语

## 申请流程

```
1. 技术交底 → 2. 专利检索 → 3. 撰写申请文件 → 4. 提交申请 → 5. 受理通知
                                                                  ↓
6. 初审合格 ← 5.1 补正（如需）                                  ↓
                                                                  ↓
7. 公布（发明18个月）                                          ↓
                                                                  ↓
8. 实质审查请求（发明3年内）                                    ↓
                                                                  ↓
9. 实审意见通知 ← 9.1 答复审查意见（可能多次）                ↓
                                                                  ↓
10. 授权/驳回                                                  ↓
                                                                  ↓
11. 缴纳年费 → 获得专利权
```

### 审查周期参考

| 专利类型 | 审查周期 |
|----------|----------|
| 发明专利 | 2-3年（可加速至6-12个月） |
| 实用新型 | 6-12个月 |
| 外观设计 | 4-6个月 |

## XML格式要求（2025年10月起）

根据国家知识产权局通知，自2025年10月1日起：

- 请求优先审查、快速审查、PPH、延迟审查、集中审查的，须以XML格式提交电子申请
- 2026年起，将仅接受XML格式的专利电子申请文件

**必要XML格式申请文件：**
- 请求书
- 权利要求书
- 说明书
- 说明书附图
- 摘要

可以使用专利业务办理系统（http://cponline.cnipa.gov.cn）中的XML格式转换工具。

## 重要提醒

1. **新颖性**：申请日前未被公开，申请后尽快提交
2. **创造性**：与现有技术相比有突出的实质性特点和显著的进步
3. **实用性**：能够制造或使用，并能产生积极效果
4. **充分公开**：说明书必须清楚、完整，使所属领域技术人员能够实施
5. **单一性**：一件专利申请限于一项发明或一个总的发明构思
6. **先申请原则**：两个以上申请人就同样的发明申请专利的，专利权授予最先申请的人
