# 中国专利申请文件详细要求

## 一、发明专利/实用新型申请文件要求

### 1.1 请求书

**必须填写的内容：**

| 项目 | 说明 |
|------|------|
| 发明名称 | 简短、准确，25字以内，不得含人名、地名、商标、型号、宣传用语 |
| 发明人 | 对发明创造的实质性特点作出创造性贡献的人 |
| 申请人 | 可以是个人或单位，可以是发明人也可以不是 |
| 联系人 | 未委托代理机构的，需指定联系人 |
| 专利代理机构 | 委托代理的，填写代理机构信息 |
| 分案申请 | 如为分案申请，需填写原申请信息 |
| 优先权 | 要求优先权的，需填写优先权信息 |
| 生物材料保藏 | 涉及生物材料的，填写保藏信息 |
| 遗传资源来源披露 | 依赖遗传资源完成的发明创造，需披露来源 |

### 1.2 说明书格式要求

**纸张要求：**
- A4纸（297mm×210mm）
- 纵向使用
- 页边距：左侧和顶部各25mm，右侧和底部各15mm

**文字要求：**
- 中文简体
- 宋体或仿宋，小四号（12pt）
- 行距1.5倍
- 字高3.5-4.5毫米
- 行距2.5-3.5毫米
- 黑色，清晰可辨

**页码要求：**
- 居中，位于每页底部
- 说明书、权利要求书、说明书附图分别编页

### 1.3 说明书各部分内容要求

#### 1.3.1 技术领域

- 写明要求保护的技术方案所属的技术领域
- 按照国际专利分类表确定
- 示例：
  - "本发明涉及通信技术领域，具体涉及一种5G网络中的信号传输方法。"
  - "本实用新型涉及机械加工设备技术领域，特别是一种具有自动定位功能的数控车床。"

#### 1.3.2 背景技术

**内容要求：**
- 写明对发明的理解、检索、审查有用的背景技术
- 引证反映背景技术的专利文献、期刊、书籍等
- 指出现有技术存在的问题和缺陷

**引证格式：**
- 专利文献：国别+公开号+公开日期
- 非专利文献：作者+文章名+期刊名+卷期+页码+出版日期

**示例：**
```
现有技术中，如中国专利CN123456789A公开了一种...，但其存在以下问题：
1. 问题一：...
2. 问题二：...
```

#### 1.3.3 发明内容

**要解决的技术问题：**
- 针对现有技术的不足，明确要解决的技术问题
- 可以列出1-3个技术问题
- 避免使用宣传用语

**技术方案：**
- 清楚、完整地描述技术方案
- 对于产品发明：描述结构、组成、连接关系
- 对于方法发明：描述步骤、流程、工艺参数

**有益效果：**
- 与现有技术相比的优点
- 可以用实验数据、对比测试等证明

#### 1.3.4 附图说明

**格式：**
```
附图说明
图1为本发明实施例1的系统结构示意图；
图2为本发明实施例1的方法流程图；
图3为本发明实施例2的装置结构示意图；
...
```

**要求：**
- 对每幅附图作简略说明
- 说明附图之间的相互关系（如有）

#### 1.3.5 具体实施方式

**内容要求：**
- 至少一个具体实施例
- 详细描述实现发明的优选方式
- 对于关键参数，给出具体数值或数值范围
- 有附图的，对照附图说明
- 对于不同实施方式，分别说明

**示例结构：**
```
【实施例1】
[详细描述实施方式，对照图1]

【实施例2】
[另一实施方式，对照图2]

【效果验证】
[实验数据或对比测试结果]
```

### 1.4 权利要求书要求

#### 1.4.1 独立权利要求

**格式（两段式）：**
```
1. 一种[产品类型]，其特征在于，包括：
    [共有技术特征A]；
    [共有技术特征B]；
    其特征是，[区别技术特征C]；
    [区别技术特征D]。
```

**内容要求：**
- 前序部分：主题名称+与现有技术共有的必要技术特征
- 特征部分："其特征是..."+区别技术特征
- 记载解决技术问题的必要技术特征

#### 1.4.2 从属权利要求

**格式：**
```
2. 根据权利要求1所述的[产品类型]，其特征在于，[附加技术特征]。

3. 根据权利要求1或2所述的[产品类型]，其特征在于，[附加技术特征]。

4. 根据权利要求1-3任一项所述的[产品类型]，其特征在于，[附加技术特征]。
```

**引用规则：**
- 只能引用在前的权利要求
- 多项从属权利要求只能以择一方式引用（用"或"）
- 多项从属权利要求不得作为另一项多项从属权利要求的基础

#### 1.4.3 权利要求撰写原则

| 原则 | 说明 |
|------|------|
| 以说明书为依据 | 技术特征必须在说明书中有记载 |
| 清楚 | 使用规范术语，边界清晰 |
| 简要 | 不得冗长，避免重复 |
| 无插图 | 可以有化学式、数学式 |
| 支持 | 每一项权利要求都能得到说明书支持 |

### 1.5 说明书附图要求

#### 1.5.1 基本要求

| 项目 | 要求 |
|------|------|
| 图号 | 图1、图2、图3... |
| 标记 | 同一组成部分标记一致 |
| 一致性 | 附图标记与文字部分一致 |
| 清晰度 | 缩小到2/3仍能清晰分辨 |
| 线条 | 黑色，均匀清晰 |

#### 1.5.2 附图类型

| 类型 | 适用情况 |
|------|----------|
| 结构示意图 | 机械装置、产品结构 |
| 流程图 | 方法、工艺步骤 |
| 电路图 | 电子电路、控制系统 |
| 方框图 | 系统组成、模块关系 |
| 曲线图 | 性能参数、实验数据 |

#### 1.5.3 附图标记规则

- 使用阿拉伯数字
- 放在相应技术特征后并置于括号内
- 不得解释为对权利要求的限制
- 示例："所述处理器（10）与存储器（20）连接..."

### 1.6 说明书摘要要求

**内容：**
- 发明名称
- 所属技术领域
- 解决的技术问题
- 技术方案要点
- 主要用途

**限制：**
- 300字以内（含标点）
- 不得使用商业性宣传用语
- 可以包含化学式

**示例：**
```
本发明公开了一种智能温控系统，属于智能家居技术领域。该系统包括温度
传感器、控制器和执行器，通过自适应PID算法实现温度的精确控制。解决了现
有温控系统响应慢、超调大的问题。可广泛应用于家用空调、冰箱等家电产品。
```

## 二、外观设计专利申请文件要求

### 2.1 图片或照片要求

**必须提交的视图：**

| 视图 | 说明 |
|------|------|
| 主视图 | 展示产品正面 |
| 后视图 | 展示产品背面 |
| 左视图 | 展示产品左侧 |
| 右视图 | 展示产品右侧 |
| 俯视图 | 展示产品顶部 |
| 仰视图 | 展示产品底部 |
| 立体图 | 展示产品三维形态（必要时） |

**图片要求：**
- 线条图或照片
- 白色或纯色背景
- 无阴影、反光、多余线条
- 比例一致
- 正投影规则绘制

### 2.2 简要说明要求

**必须写明的内容：**

| 项目 | 说明 |
|------|------|
| 产品名称 | 符合国际外观设计分类表 |
| 用途 | 产品的用途 |
| 设计要点 | 形状、图案、色彩的结合 |
| 省略视图 | 说明省略的视图及原因 |
| 色彩保护 | 请求保护色彩的，声明并指定色标 |
| 透明材料 | 由透明材料制成的，注明 |
| 细长物品 | 细长物品可省略部分长度 |

**示例：**
```
1. 本外观设计产品的名称：智能手表。
2. 本外观设计产品的用途：用于佩戴在手腕上，显示时间、
   接收通知等。
3. 本外观设计的设计要点：在于产品的形状。
4. 最能表明设计要点的图片或照片：立体图。
5. 省略视图：仰视图无设计要点，故省略。
6. 请求保护的外观设计包含色彩。
```

## 三、电子申请格式要求

### 3.1 文件格式

| 文件类型 | 格式要求 |
|----------|----------|
| 请求书 | XML（2025.10起逐步要求） |
| 说明书 | XML/PDF |
| 权利要求书 | XML/PDF |
| 说明书附图 | XML/TIF/PDF |
| 摘要 | XML/PDF |
| 证明文件 | PDF |

### 3.2 XML格式要求（2025年10月起）

**必须使用XML格式的情形：**
- 请求优先审查
- 请求快速审查
- 请求专利审查高速路（PPH）
- 请求延迟审查
- 请求集中审查

**转换工具：**
- 专利业务办理系统（http://cponline.cnipa.gov.cn）
- "工具下载"栏目提供XML格式转换工具

### 3.3 文件命名规范

| 文件内容 | 文件名 |
|----------|--------|
| 请求书 | 100001.xml / request.pdf |
| 说明书 | 100002.xml / description.pdf |
| 权利要求书 | 100003.xml / claims.pdf |
| 说明书附图 | 100004.tif / drawings.pdf |
| 说明书摘要 | 100005.xml / abstract.pdf |
| 摘要附图 | 100006.tif / abstract_drawing.tif |

## 四、特殊领域要求

### 4.1 涉及生物材料的申请

**保藏要求：**
- 申请日前或申请日提交样品至保藏单位
- 申请日起4个月内提交保藏证明和存活证明
- 说明书中记载保藏信息

**保藏单位：**
- 中国典型培养物保藏中心（CCTCC）
- 中国微生物菌种保藏管理委员会普通微生物中心（CGMCC）

### 4.2 涉及遗传资源的申请

**披露要求：**
- 直接来源：获取该遗传资源的直接渠道
- 原始来源：该遗传资源的原生境所在地
- 如无法说明原始来源，需陈述理由

### 4.3 涉及计算机程序的发明

**可专利性：**
- 为解决技术问题，利用技术手段，实现技术效果
- 例如：提高设备性能、改善数据处理效率、增强系统安全性等

**撰写要点：**
- 以方法权利要求或装置权利要求形式保护
- 详细描述算法流程、数据处理步骤
- 提供程序流程图
- 说明技术效果

### 4.4 涉及化学发明的申请

**充分公开要求：**
- 化学产品：明确化学名称、分子式、结构式、制备方法
- 化学方法：详细工艺条件（温度、压力、时间、催化剂等）
- 实施例：证明发明可实现，数据完整

**权利要求类型：**
- 产品权利要求：化合物、组合物、制品
- 方法权利要求：制备方法、使用方法、处理方法
- 用途权利要求：新产品的新用途
