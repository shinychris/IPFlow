# 中国专利申请文件模板

## 一、说明书模板

### 1.1 发明名称

要求：
- 25字以内
- 不得含人名、地名、商标、型号
- 不得使用宣传用语
- 与请求书中名称一致

**产品发明示例：**
- "一种智能温控装置"
- "一种基于深度学习的图像识别系统"
- "一种可降解环保材料及其制备方法"

**方法发明示例：**
- "一种数据加密传输方法"
- "一种锂电池快速充电方法"
- "一种蛋白质分离纯化方法"

### 1.2 说明书正文模板

```markdown
[发明名称]

技术领域
本发明涉及[技术领域]技术领域，具体涉及一种[具体技术内容]。

背景技术
[简要介绍该技术领域的现有技术状况]

目前，现有技术中存在以下问题：
[问题1的描述]，如中国专利CNxxxxxxxxxA公开的[现有技术描述]，该技术[存在的问题]。

[问题2的描述]，...

因此，亟需一种[本发明要解决的技术问题]的[产品/方法]。

发明内容
本发明旨在至少解决现有技术中存在的技术问题之一。为此，本发明提出一种
[发明名称]，能够[主要技术效果]。

根据本发明的第一方面，提供一种[产品/方法]，包括：
[详细描述技术方案，包括各部分结构、连接关系、工作方式等]

根据本发明的一些实施例，[可选的附加技术特征]。

根据本发明的一些实施例，[可选的附加技术特征]。

根据本发明的第二方面，提供一种[另一技术方案，如适用]。

本发明实施例的有益效果包括：
[有益效果1]，例如：...
[有益效果2]，例如：...
[有益效果3]，例如：...

附图说明
图1为本发明实施例1的[装置/方法]的[结构/流程]示意图；
图2为本发明实施例1的[某一组件]的结构示意图；
图3为本发明实施例2的[变化实施方式]的示意图；
图4为[实验结果/效果对比]图。

具体实施方式
下面将结合本发明实施例中的附图，对本发明实施例中的技术方案进行清楚、
完整地描述，显然，所描述的实施例仅仅是本发明一部分实施例，而不是全部的
实施例。基于本发明中的实施例，本领域普通技术人员在没有做出创造性劳动前
提下所获得的所有其他实施例，都属于本发明保护的范围。

在本发明的描述中，需要理解的是，术语"中心"、"纵向"、"横向"、"长度"、
"宽度"、"厚度"、"上"、"下"、"前"、"后"、"左"、"右"、"竖直"、"水平"、
"顶"、"底"、"内"、"外"、"顺时针"、"逆时针"等指示的方位或位置关系为基于
附图所示的方位或位置关系，仅是为了便于描述本发明和简化描述，而不是指示
或暗示所指的装置或元件必须具有特定的方位、以特定的方位构造和操作，因此
不能理解为对本发明的限制。

实施例1
[详细描述第一实施方式，包括：]
[1. 各组成部分的详细描述]
[2. 各部分的连接关系]
[3. 工作原理说明]
[4. 对照图1的详细说明]

在本实施例中，所述[组件A]（10）包括[具体描述]。所述[组件A]（10）与所述
[组件B]（20）通过[连接方式]连接，...

当[产品/系统]工作时，[详细描述工作过程]，...

实施例2
[描述与实施例1不同的另一种实施方式]

与实施例1相比，本实施例的区别在于[区别点]，...

[效果验证]
为验证本发明的技术效果，进行了以下实验：

实验1：[实验名称]
实验条件：...
实验结果：...

实验2：[实验名称]
实验条件：...
实验结果：...

[对比实验结果，证明本发明的有益效果]

虽然在上文中已经参考一些实施例对本发明进行了描述，然而在不脱离本发明
的范围的情况下，可以对其进行各种改进并且可以用等效物替换其中的部件。尤
其是，只要不存在结构冲突，本发明所披露的各个实施例中的各项特征均可通过
任意方式相互结合起来使用，在本说明书中未对这些组合的情况进行穷举性描述
仅仅是出于省略篇幅和节约资源的考虑。因此，本发明并不局限于文中公开的特
定实施例，而是包括落入权利要求的范围内的所有技术方案。
```

### 1.3 不同类型发明的撰写重点

#### 机械装置类

```markdown
技术方案描述重点：
1. 整体结构组成
2. 各部件的形状、位置、连接关系
3. 运动关系和动力传递
4. 工作过程描述

附图要求：
- 整体结构示意图
- 关键部件的详细结构图
- 装配关系图
- 工作原理示意图
```

#### 电子电路类

```markdown
技术方案描述重点：
1. 系统整体架构（方框图）
2. 各功能模块的电路原理
3. 信号流程和数据处理过程
4. 元器件的参数选择

附图要求：
- 系统方框图
- 详细电路原理图
- 信号时序图
- 关键节点的波形图
```

#### 计算机程序类

```markdown
技术方案描述重点：
1. 系统整体架构
2. 各模块的功能和接口
3. 算法流程和逻辑
4. 数据处理步骤

附图要求：
- 系统架构图
- 模块连接关系图
- 程序流程图
- 数据结构示意图

权利要求撰写方式：
- 方法权利要求：描述步骤流程
- 装置权利要求：描述功能模块
```

#### 化学配方/工艺类

```markdown
技术方案描述重点：
1. 组成成分及各组分含量/比例
2. 制备工艺的详细步骤
3. 工艺参数（温度、压力、时间、pH等）
4. 反应条件控制

实施例要求：
- 至少3个实施例
- 详细给出各组分的具体用量
- 完整的制备步骤
- 性能测试数据
```

## 二、权利要求书模板

### 2.1 产品发明权利要求书模板

```markdown
权利要求书

1. 一种[产品名称]，其特征在于，包括：
    [部件A]（标记）；
    [部件B]（标记），所述[部件B]与所述[部件A][连接关系]；
    [部件C]（标记），所述[部件C]用于[功能描述]。

2. 根据权利要求1所述的[产品名称]，其特征在于，所述[部件A]包括：
    [子部件A1]；
    [子部件A2]，所述[子部件A2]与所述[子部件A1][关系]。

3. 根据权利要求1或2所述的[产品名称]，其特征在于，所述[部件B]
    为[具体限定]。

4. 根据权利要求1-3任一项所述的[产品名称]，其特征在于，还包括：
    [附加部件D]。

5. 根据权利要求1所述的[产品名称]，其特征在于，所述[部件A]与
    所述[部件B]通过[具体连接方式]连接。

6. 根据权利要求1所述的[产品名称]，其特征在于，所述[部件C]设置
    在[具体位置]。

[继续添加从属权利要求，对各个技术特征进行进一步限定]
```

### 2.2 方法发明权利要求书模板

```markdown
权利要求书

1. 一种[方法名称]，其特征在于，包括以下步骤：
    获取[数据/信息]；
    对所述[数据]进行[处理步骤A]；
    根据[处理步骤A]的结果，执行[处理步骤B]；
    输出[结果]。

2. 根据权利要求1所述的[方法名称]，其特征在于，所述对所述[数据]
    进行[处理步骤A]包括：
    对所述[数据]进行[子步骤A1]；
    将经过[子步骤A1]处理的[数据]进行[子步骤A2]。

3. 根据权利要求1或2所述的[方法名称]，其特征在于，所述获取[数据]
    包括：
    从[数据源]接收原始数据；
    对所述原始数据进行预处理。

4. 根据权利要求1所述的[方法名称]，其特征在于，所述[处理步骤A]
    采用[具体算法/技术]。

5. 根据权利要求1所述的[方法名称]，其特征在于，所述[处理步骤B]
    在[条件]时执行。

6. 一种[系统/装置]，其特征在于，包括：
    获取模块，用于获取[数据]；
    处理模块，用于对所述[数据]进行[处理]；
    输出模块，用于输出[结果]。

7. 一种计算机可读存储介质，其上存储有计算机程序，其特征在于，
    该计算机程序被处理器执行时实现如权利要求1-5任一项所述的
    [方法名称]。

8. 一种电子设备，包括存储器、处理器及存储在存储器上并可在处理器
    上运行的计算机程序，其特征在于，所述处理器执行所述程序时实现
    如权利要求1-5任一项所述的[方法名称]。
```

### 2.3 权利要求布局策略

```
权利要求1（独立权利要求）
├── 权利要求2（从属，限定A）
├── 权利要求3（从属，限定B）
│   └── 权利要求5（从属3，进一步限定B）
├── 权利要求4（从属，限定C）
└── 权利要求6（从属，增加D）

权利要求7（独立权利要求，另一技术方案）
├── 权利要求8（从属7）
└── 权利要求9（从属7）

权利要求10（产品权利要求，方法对应的装置）
```

## 三、说明书摘要模板

### 3.1 摘要模板

```markdown
说明书摘要

本发明公开了一种[发明名称]，属于[技术领域]技术领域。该[产品/方法]
包括[主要组成部分/主要步骤]，通过[核心技术手段]，解决了[现有技术存在
的问题]，实现了[主要技术效果]。本发明[另一优点]，可广泛应用于[应用
领域]。
```

### 3.2 摘要附图指定

在请求书中指定最能说明发明技术特征的附图作为摘要附图。

## 五、外观设计申请文件模板

### 5.1 简要说明模板

```markdown
简要说明

1. 本外观设计产品的名称：[产品名称]。

2. 本外观设计产品的用途：用于[具体用途描述]。

3. 本外观设计的设计要点：在于[形状/图案/形状与图案的结合/
   形状与色彩的结合/图案与色彩的结合/形状、图案与色彩的结合]。

4. 最能表明设计要点的图片或照片：[立体图/主视图等]。

5. 省略视图：[俯视图/仰视图/左视图等]无设计要点，故省略。

6. [如请求保护色彩]本外观设计请求保护色彩。

7. [如为平面产品]本外观设计为平面产品，单元图案四方连续
   无限定边界。
```

### 5.2 图片拍摄/绘制要求

```markdown
图片/照片要求：

1. 背景：纯色背景（推荐白色），无干扰物
2. 光线：均匀照明，无阴影、反光
3. 视角：正投影，无透视变形
4. 比例：六面视图比例一致
5. 清晰度：分辨率300dpi以上
6. 色彩：如请求保护色彩，提交彩色图片/照片
```
