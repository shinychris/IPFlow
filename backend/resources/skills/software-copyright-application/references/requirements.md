# 软件著作权申请材料详细要求

## 一、源程序代码要求

### 1.1 基本规定

| 项目 | 要求 |
|------|------|
| 提交方式 | 一般交存（前30页+后30页）或例外交存（需说明理由） |
| 总页数 | 60页（前30页 + 后30页） |
| 不足60页 | 若源代码总量不足60页，需提交全部源代码 |
| 每页行数 | 不少于50行（不含空行） |
| **文件格式** | **.docx**（本技能生成；用户最终提交中心时再转 PDF） |
| **注释率** | **<5%（硬性指标，详见 anti-ai-detection.md）** |

### 1.2 格式规范（docx）

**页面设置：**
- **纸张：A4（210×297mm）** —— 必须为 A4，不得用 Letter
- 页边距：上下 1"（2.54cm），左右 0.8–1.25"（约 2.0–3.17cm）
- 打印方式：单面打印

**字体和段落（代码文件特有）：**
- **代码字体：Consolas / Courier New（等宽字体）** —— 不得用宋体
- 字号：五号（10.5pt）
- 行距：单倍
- 对齐：左对齐

**页眉页脚：**
- **页眉内容：软件全称 + 版本号**（所有页面显示，如"销售通智能销售助手软件 V1.0"）
- 页眉格式：居中，小五号字
- **页码：底部居中，连续编排**（多文件合并时 1–30 / 31–60 连续）
- 页码格式：纯数字（1, 2, 3...）

**示例页眉：**
```
                    XX管理系统 V1.0
                              （页码在页脚底部居中）
```

### 1.3 内容要求

**前30页选择：**
1. 项目配置文件（package.json, pom.xml, build.gradle等）
2. 主入口文件（main.py, App.java, index.js等）
3. 核心模块初始化代码
4. 数据模型/实体类定义
5. 公共工具类/基础类

**后30页选择：**
1. 核心业务逻辑实现
2. 关键算法代码
3. 重要功能模块实现
4. 数据处理/业务规则代码
5. 确保最后一个模块完整结束

**代码处理：**
- 删除所有空行（使用Word替换：`^p^p` → `^p`，重复至无匹配）
- 删除敏感信息（密码、密钥、API Key、测试账号等）
- **注释率严格 <5%**：删除一切显而易见的注释，仅保留复杂算法/关键业务规则
- 每个源文件前注明「代码文件: <相对路径>」
- **message 中文化**：代码中 message/提示/异常文案若为英文，必须改成中文（如 `'User not found'` → `'用户不存在'`）
- **注释禁英文**：注释除函数名/类名/专业术语外禁止英文，必须用中文
- 代码必须真实，不得伪造

### 1.4 不同语言注意事项

| 语言 | 注意事项 |
|------|----------|
| Java | 保留package/import语句，类定义完整 |
| Python | 保留import语句，注意缩进格式 |
| JavaScript/TypeScript | 保留require/import，函数定义完整 |
| C/C++ | 保留#include，头文件定义完整 |
| Go | 保留package/import，函数签名完整 |
| 前端Vue/React | 保留模板和脚本部分 |

## 二、软件文档要求

### 2.1 文档类型

可任选以下一种：
1. **用户手册** - 适合有界面的应用软件
2. **操作手册** - 适合工具类软件
3. **设计说明书** - 适合系统类、无界面软件

### 2.2 基本规定

| 项目 | 要求 |
|------|------|
| **文件格式** | **.docx**（本技能生成；用户最终提交中心时再转 PDF） |
| 提交页数 | 前30页 + 后30页 = 60页 |
| 不足60页 | 若文档不足60页，提交全部内容 |
| 每页行数 | 不少于30行（含图片页面除外） |
| 图文要求 | 建议图文结合，软件需配界面截图 |

### 2.3 格式规范（docx）

**页面设置：**
- **纸张：A4（210×297mm）**
- 页边距：上下 1"（2.54cm），左右 0.8–1.25"
- 打印方式：单面打印

**字体和段落（说明书通用规范）：**
- **正文字体：宋体，小四（12pt）**
- **标题字体：黑体**（三号/四号/小四，根据级别）
- 行距：1.5倍行距或固定值20磅
- 首行缩进：2字符

**页眉页脚：**
- **页眉内容：软件全称 + 版本号**（所有页面显示）
- 页眉格式：居中，小五号字
- **页码：底部居中，连续编排**
- 页码格式：纯数字

### 2.4 用户手册推荐结构

```
封面
├── 软件名称
├── 版本号
└── 编制日期

目录（可选，但建议包含）

第1章 软件概述
├── 1.1 软件简介（200-300字）
├── 1.2 主要功能
├── 1.3 运行环境
│   ├── 硬件环境
│   └── 软件环境
└── 1.4 软件安装

第2章 功能使用说明
├── 2.1 [功能模块一名称]
│   ├── 2.1.1 功能描述
│   ├── 2.1.2 操作步骤（配截图）
│   └── 2.1.3 注意事项
├── 2.2 [功能模块二名称]
...
└── 2.n [功能模块n名称]

第3章 系统管理（如适用）
├── 3.1 用户管理
├── 3.2 权限设置
├── 3.3 数据备份
└── 3.4 日志查看

第4章 常见问题解答
└── Q&A列表

附录（可选）
├── 附录A 术语表
├── 附录B 快捷键列表
└── 附录C 联系方式
```

### 2.6 截图要求

- **清晰度**：界面文字清晰可见
- **完整性**：显示完整窗口（含标题栏）
- **标注**：关键操作步骤用红色框或箭头标注
- **数量**：每个主要功能配1-2张截图
- **格式**：JPG或PNG，**嵌入 docx 文档中**（不外挂）

## 三、身份证明文件要求

### 3.1 个人申请

| 项目 | 要求 |
|------|------|
| 文件类型 | 身份证正反面复印件 |
| 签字要求 | 在复印件空白处手写签名 |
| 注明用途 | 建议注明"仅用于软件著作权登记" |
| 格式 | PDF扫描件 |

### 3.2 企业申请

| 项目 | 要求 |
|------|------|
| 文件类型 | 营业执照副本复印件 |
| 盖章要求 | 加盖企业公章（鲜章） |
| 清晰度 | 公章和文字清晰可辨 |
| 格式 | PDF扫描件 |

### 3.3 事业单位/机关/社团

| 类型 | 所需文件 |
|------|----------|
| 事业单位 | 事业单位法人证书副本复印件（盖章） |
| 机关法人 | 机关法人证书副本复印件（盖章） |
| 社团法人 | 社团法人登记证书复印件（盖章） |

## 四、权利归属证明文件

### 4.1 合作开发

- 提交《合作开发协议》或《合作开发合同》
- 协议需明确各方权利归属比例
- 所有著作权人签字/盖章

### 4.2 委托开发

- 提交《委托开发合同》
- 合同中需明确著作权归属条款
- 双方签字/盖章

### 4.3 职务开发

- 提交《劳动合同》或《开发任务书》
- 或提交《非职务软件开发保证书》（个人申请时）

### 4.4 版本说明（V1.0以上版本）

- 提交《版本说明》
- 说明新版本相比旧版本的改进内容
- 著作权人签字/盖章

## 五、材料一致性要求

### 5.1 必须一致的字段

| 字段 | 说明 |
|------|------|
| 软件全称 | 申请表、源代码、说明书、身份证明中的名称必须完全一致 |
| 版本号 | V的大小写、数字必须一致 |
| 著作权人 | 名称需与证件完全一致 |
| 开发完成日期 | 所有材料中日期逻辑一致 |

### 5.2 一致性检查清单

- [ ] 申请表软件名称 = 源代码页眉软件名称
- [ ] 申请表软件名称 = 说明书页眉软件名称
- [ ] 申请表版本号 = 源代码页眉版本号
- [ ] 申请表版本号 = 说明书页眉版本号
- [ ] 申请表著作权人 = 身份证明文件名称
- [ ] 申请表代码行数 ≈ 实际提交代码行数（误差合理）
- [ ] 说明书描述功能 ⊆ 源代码实现功能
