# 软著申请常见问题与避坑指南

## 一、常见补正原因

### 1.1 格式问题（占比最高）

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 代码行数不足 | 每页少于50行 | 调整行距或字号，确保每页≥50行 |
| 文档行数不足 | 每页少于30行 | 增加内容或调整行距 |
| 缺少页眉 | 代码/文档页眉未标注软件信息 | 添加页眉：软件名称+版本号 |
| 页码不连续 | 页码中断或重复 | 检查分节符设置，确保页码连续 |
| 有封面/目录 | 源代码/文档开头有封面页 | 删除封面和目录，从正文开始 |

### 1.2 一致性问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 软件名称不一致 | 不同材料中名称有差异 | 统一所有材料中的软件名称 |
| 版本号不一致 | V的大小写不一致 | 统一为V1.0格式 |
| 代码行数不符 | 申请表填写与实际不符 | 准确统计代码行数后填写 |
| 开发日期矛盾 | 早于公司成立日期 | 核实并修正开发完成日期 |

### 1.3 内容问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 代码与功能不符 | 说明书描述的功能代码中没有 | 确保说明书描述的是已实现功能 |
| 代码不真实 | 添加大量无意义注释凑行数 | 删除无意义内容，提交真实代码 |
| 截图不清晰 | 界面截图模糊或不完整 | 重新截取清晰完整的界面图 |
| 文档过于简单 | 说明书只有几页 | 补充详细的功能说明和操作步骤 |

### 1.4 签章问题

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 未签字/盖章 | 申请表缺少签名或公章 | 手写签名或加盖鲜章 |
| 使用电子章 | 使用了PS的电子章 | 使用真实公章或手写签名 |
| 复印件未签字 | 身份证复印件没有手写签名 | 在空白处手写签名 |
| 公章不清晰 | 扫描件公章模糊 | 重新扫描，确保公章清晰 |

## 二、避坑指南

### 2.1 软件命名避坑

**禁用词汇（不得使用）：**
- 含有"中国"、"国家"、"中央"、"全国"等字样
- 含有"国家级"、"最高级"、"最佳"等绝对化用语
- 与他人已登记软件名称高度相似
- 过于宽泛或通用的名称（如"管理系统"）

**命名建议：**
- ✅ XX智能办公管理系统软件
- ✅ XX在线教育平台软件
- ✅ XX数据分析工具软件
- ✅ XX电商导购APP软件
- ❌ 中国最好的管理系统
- ❌ 国家级安全软件
- ❌ 管理系统（过于宽泛）

### 2.2 代码整理避坑

**不要做的事：**
- ❌ 添加大量无意义的注释（如连续几十行"//"或"/**/"）
- ❌ 复制粘贴重复代码凑行数
- ❌ 保留包含密码、密钥、测试账号的代码
- ❌ 提交空行过多的代码
- ❌ 使用图片代替代码

**应该做的事：**
- ✅ 删除所有空行（Word中替换：`^p^p` → `^p`）
- ✅ 删除或脱敏敏感信息
- ✅ 保留核心逻辑和结构
- ✅ 确保代码真实有效
- ✅ 代码格式统一，缩进规范

### 2.3 说明书编写避坑

**常见问题：**
- ❌ 说明书只有几页，内容过于简略
- ❌ 截图不清晰或缺少关键界面
- ❌ 功能描述与技术方案脱节
- ❌ 只描述功能，没有操作步骤

**编写建议：**
- ✅ 图文结合，每个功能配截图
- ✅ 详细描述操作步骤
- ✅ 功能描述与代码实现对应
- ✅ 确保每页不少于30行（图片页除外）

### 2.4 申请表填写避坑

**易错字段：**

| 字段 | 常见错误 | 正确做法 |
|------|----------|----------|
| 版本号 | v1.0、1.0、V1 | 统一为 V1.0 |
| 开发完成日期 | 早于公司成立日期 | 核实公司注册时间 |
| 代码行数 | 随便填写 | 实际统计后填写 |
| 软件功能 | 写得太简单或太复杂 | 300字内概括主要功能 |
| 硬件环境 | 写得太高或太低 | 根据实际需求填写 |

### 2.5 提交流程避坑

**重要提醒：**
1. **先实名认证再申请**：所有著作权人需提前完成实名认证（约3个工作日）
2. **检查材料完整性**：提交前对照材料清单检查
3. **PDF格式**：所有材料需转换为PDF格式
4. **文件大小**：注意单个文件大小限制（一般不超过30MB）
5. **备份材料**：保留所有材料的原件和电子版

## 三、审核周期与进度查询

### 3.1 审核周期

| 类型 | 受理时间 | 审查周期 | 总周期 |
|------|----------|----------|--------|
| 普通登记 | 1-2个月 | 30-60个工作日 | 2-4个月 |

**注意：** 以上时间为参考，实际周期可能因申请量波动。

### 3.2 进度查询

1. 登录中国版权保护中心官网
2. 进入"我的登记" → "软件登记"
3. 查看申请状态：
   - **待受理**：材料已提交，等待受理
   - **受理中**：已受理，正在审查
   - **补正通知**：需要修改材料
   - **已批准**：审查通过，等待发证
   - **已发证**：证书已发放

### 3.3 补正处理

**收到补正通知后：**
1. 仔细阅读补正意见
2. 按要求修改相关材料
3. 在系统中上传修改后的材料
4. 等待重新审查

**注意：** 补正有时间限制（通常30天内），超期视为撤回申请。

## 四、特殊情况处理

### 4.1 合作开发

**必须提供：**
- 合作开发协议（所有著作权人签字/盖章）
- 所有著作权人的身份证明
- 所有著作权人完成实名认证

**协议要点：**
- 明确各方权利归属比例
- 明确软件名称和版本
- 明确开发内容和分工

### 4.2 委托开发

**必须提供：**
- 委托开发合同
- 合同中需明确著作权归属条款

**注意：** 如无明确约定，著作权归受托方（开发方）所有。

### 4.3 职务开发

**企业申请个人开发的软件：**
- 提供劳动合同或开发任务书
- 或提供《非职务软件开发保证书》（个人出具）

### 4.4 版本升级（V1.0以上）

**需要额外提供：**
- 《版本说明》：说明新版本相比旧版本的改进
- 原软件著作权证书（如有）

**注意：** 版本升级是重新申请，不是变更登记。

### 4.5 代码不足60页

**处理办法：**
- 若总代码量不足60页，提交全部代码
- 申请表中如实填写代码总行数
- 在代码文档末尾注明"以上为全部源代码"

## 五、实用技巧

### 5.1 快速统计代码行数

**Linux/Mac：**
```bash
# 统计某类文件行数
find . -name "*.java" -exec wc -l {} +

# 统计多种类型文件
find . -type f \( -name "*.java" -o -name "*.xml" \) -exec wc -l {} +

# 排除某些目录
find . -type f -name "*.java" ! -path "*/test/*" -exec wc -l {} +
```

**Windows：**
```powershell
# PowerShell统计
gci -r -filter "*.java" | Get-Content | Measure-Object
```

**IDE插件：**
- IntelliJ IDEA: Statistic 插件
- VS Code: Code Counter 插件

### 5.2 Word快速去空行

1. Ctrl+H 打开替换对话框
2. 查找内容输入：`^p^p`（两个段落标记）
3. 替换为输入：`^p`（一个段落标记）
4. 点击"全部替换"
5. 重复步骤2-4，直到提示"替换0处"

### 5.3 页眉页码设置技巧

**分节符使用：**
- 如果需要在不同部分使用不同页眉，使用"下一页"分节符
- 取消"链接到前一节"选项

**页码格式：**
- 起始页码设为1
- 使用普通数字格式
- 位置：页面顶端右侧

### 5.4 PDF转换注意事项

- 使用"另存为PDF"功能，不要用虚拟打印
- 确保嵌入所有字体
- 检查PDF中的链接和图片是否正常
- 压缩PDF大小（如有必要），但保证清晰度
